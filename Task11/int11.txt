1Ô∏è‚É£ What are Regular Expressions (Regex)?
Interview Answer

Regular Expressions (regex) are patterns used to search, match, extract, or validate text based on specific rules.

Why regex exists

To avoid writing long conditional logic

To handle complex string patterns efficiently

To validate inputs like email, phone number, password, etc.

Simple Example (Python)
import re

pattern = r'\d+'
text = "Order ID is 12345"

result = re.search(pattern, text)
print(result.group())


Output:

12345


Here:

\d+ ‚Üí matches digits

Regex finds numbers inside text easily

2Ô∏è‚É£ Difference between match, search, and findall
Interview Answer

These methods differ in where and how they search for a pattern.

Comparison Table

| Method      | Description                                  |
| ----------- | -------------------------------------------- |
| `match()`   | Checks match **only at beginning** of string |
| `search()`  | Finds first match **anywhere** in string     |
| `findall()` | Returns **all matches** as a list            |


Example
import re

text = "My number is 9876543210"

print(re.match(r'\d+', text))       # None
print(re.search(r'\d+', text))      # Match object
print(re.findall(r'\d+', text))     # ['9876543210']

Key Interview Line

match() is strict, search() is flexible, and findall() is exhaustive.

Interview Answer

Regex is preferred for validation because it allows complex pattern checking in a single line, making code cleaner, faster, and more reliable.

Why not normal if-else?

Multiple conditions ‚Üí messy code

Hard to maintain

Error-prone

‚úÖ With Regex:

Interview Advantage

4Ô∏è‚É£ What are Greedy and Non-Greedy Patterns?
Interview Answer

Greedy patterns match as much text as possible, while
non-greedy (lazy) patterns match as little text as possible.

üîπ Greedy Example
import re

text = "<tag>content</tag>"
print(re.search(r'<.*>', text).group())


Output:

<tag>content</tag>


‚û° .* eats everything

üîπ Non-Greedy Example
print(re.search(r'<.*?>', text).group())


Output:

<tag>


‚û° .*? stops at first match

Interview One-Liner

Adding ? makes a greedy quantifier non-greedy.
